## What's New in v3.0.0

### New Instruments

- **JWST/NIRCam** ŌĆö Full instrument class with PSF caching, NIRCam filter response curves (F115W, F150W, F200W, F277W, F356W, F444W), throughput data, and GalSim rendering support.

### romanisim Pipeline

- Initial version of new `romanisim` engine for end-to-end Roman Space Telescope Level 1 / Level 2 / Level 3 simulation using the official `romanisim` instrument simulator and `romancal` mosaic pipeline.
- Detector-bucketed PSF caching for romanisim FFIs using CRDS.

### Pipeline Features

- `--resume` flag (replaces `--force`) to continue interrupted pipeline runs without reprocessing completed steps.
- `--sequential` flag for in-order (non-parallelized) processing, useful for debugging.
- Labeled / unlabeled dataset mode: answer key generation for unlabeled exports, driven by config.
- Corrupted pickle scanner utility to detect bad intermediate files.
- `limit` support in survey simulation steps.

### Science Features

- **Lensed supernovae** ŌĆö simulate lensed SNe from SLSim sources including substructure; produce time-series exposures.
- **COSMOS-Web sources** ŌĆö band-specific image sources via SLSim catalog integration with per-band remapping.
- **Mass sheet correction** ŌĆö optional mass sheet degeneracy correction in `add_realization`.
- **Photometry utilities** ŌĆö flux unit conversions (cps Ōåö magnitudes Ōåö maggies), magnitude zeropoint stored on `SyntheticImage`, sky background in cps as a passable argument.
- **Lens subtraction** ŌĆö routine to subtract the lensing galaxy from simulated images.
- **Smooth NaN / negative pixels** ŌĆö methods on `Exposure` to clean artefacts.
- **Subhalo overplot** ŌĆö improved `overplot_subhalos` method.
- `add_realization` convenience method for adding PyHalo realizations directly.
- `SyntheticImage` now tracks computation time.

### Logging & Configuration

- Full `logging` implementation throughout the pipeline (configurable level in `mejiro` config YAML).
- New YAML configuration files for `all`, `hwo`, `jwst`, and `roman_data_challenge` (rung 0 / rung 1) pipelines.
- `mejiro` configuration file documented.

### Testing

- Expanded unit tests: `cosmo`, `snr_calculation`, HST, LSST, `util`, `strong_lens`, `roman_utils`.
- Shared pytest fixtures across test suite.

### Breaking Changes

- **`SyntheticImage.image` renamed to `.data`** and **`Exposure.exposure` renamed to `.data`** ŌĆö update any direct attribute access.
- **Python Ōēź 3.13** required (raised from 3.10).
